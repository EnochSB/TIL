# 관계형 데이터베이스 용어 정리
## Table (aka Relation)
- 데이터가 들어있는 곳

## Field (aka Column, Attribute)
- 특정 종류(타입)의 데이터 그룹
- 표의 열(column)에 해당

## Record (aka Row, Tuple)​
- 구체적인 데이터 값
- 표의 행(row)에 해당

## Database (aka Schema)
- 테이블의 집합

## Primary Key(PK)
- 각 레코드를 구분하기 위한 고유한 값
- 관계형 데이터베이스에서 레코드 식별자로 활용된다.

## Foreign Key(FK)
- 다른 테이블의 레코드를 사용하기 위해 갖고오는 키
- 서로 다른 테이블 간에 관계를 만들기 위해 사용된다.
